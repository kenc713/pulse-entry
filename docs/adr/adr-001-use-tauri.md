# ADR-001: Use Tauri for Desktop Application

## Status

Accepted

## Context

**Problem:**

- pulse-entry は Windows デスクトップアプリケーション
- ユーザーデータ（気分 + 仕事内容）を Excel に蓄積
- 指定時間（XX:00）に定期実行でポップアップを表示

**Candidates:**

1. **Electron**
   - メリット: JavaScript のみ、学習コスト低い、情報豊富
   - デメリット: メモリ消費 150-200MB、アプリサイズ 180-250MB
2. **Tauri**
   - メリット: メモリ効率（30-60MB 使用）、小さいサイズ（40-50MB）、シンプルな設計
   - デメリット: Rust 学習が必要（ただし OOP 経験者なら 1-2週間）

## Decision

**Tauri を採用する**

Rust は OOP 経験者にとって学習曲線が思ったより緩く、2-3週間で MVP に必要な基本概念（所有権・借用・構造体）を習得できる。

メモリ効率とシンプルな設計による長期的な保守性が、初期学習コストを上回る価値を提供する。

## Rationale

- **シンプルなロジック**（Excel への書き込み）に見合った環境
- **OOP 経験がある**ため、Rust 習得は「言語学習」だけに絞れる
- **2-3週間の投資** により、将来の Rust プロジェクトにも活かせるスキル獲得
- **デスクトップアプリとしての品質**：メモリ効率、起動速度、デプロイサイズ

## Consequences

**利点:**

- ユーザーマシンへの負荷が少ない
- アプリサイズが小さい（ダウンロード・インストールが軽い）
- システムプログラミング言語としての Rust スキルが蓄積

**学習・開発:**

- Week 1: Rust 基本概念（所有権、借用、ライフタイム）
- Week 2: Tauri API + フロント連携
- Week 3: MVP 実装

**リスク:**

- Rust エコシステムへの新規参入
- Stack Overflow 日本語情報が限定的（英語が必須）

## Related Decisions

- データストレージ: Excel + Interface で DI (将来的に DB 乗り換え可能)
- UI フレームワーク: Vue.js (フロント側)
- スケジューリング: Windows タスクスケジューラ or Tauri ネイティブ
